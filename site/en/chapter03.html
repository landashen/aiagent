Chapter 3: Key Technologies for AI Agents

"Illustrated AI Agent" Series - Chapter 3

3.1 Large Language Models (LLM) as the Agent Brain

LLMs serve as the cognitive core of AI Agents, providing natural language understanding, reasoning, code generation, and multi-turn dialogue capabilities. This section covers how different LLMs (GPT-4, Claude, Gemini, open-source models) compare as Agent brains, key performance factors like context window size and reasoning depth, and strategies for selecting and optimizing LLMs for specific Agent use cases.

"Illustration: LLM as the Agent Engine — Powering Understanding, Reasoning, Planning, Generation"

3.2 Prompt Engineering for Agents

Prompt Engineering is crucial for Agent performance, covering system prompts that define Agent identity and behavior boundaries, few-shot examples that guide tool usage patterns, structured output formats (JSON, XML) for reliable tool interaction, and advanced techniques like role-playing prompts, chain-of-thought prompts, and meta-prompts that enable Agents to dynamically construct their own prompts.

3.3 ReAct Framework: Interleaving Reasoning and Action

ReAct (Reasoning + Acting) is one of the most influential Agent frameworks. It alternates between thinking steps (Thought), action steps (Action), and observation steps (Observation) to create a transparent, controllable decision loop. This approach reduces hallucination by grounding reasoning in real observations and enables complex multi-step tasks. The section compares ReAct with pure reasoning and pure acting approaches.

3.4 Chain-of-Thought (CoT) Reasoning

Chain-of-Thought prompting enables Agents to solve complex problems by breaking them into explicit intermediate reasoning steps. Variants include zero-shot CoT ("Let's think step by step"), few-shot CoT with worked examples, self-consistency (sampling multiple reasoning paths), and Tree of Thoughts for branching exploration. CoT dramatically improves performance on mathematical, logical, and multi-step planning tasks.

3.5 Function Calling & Tool Invocation

Function Calling standardizes how LLMs interact with external tools by generating structured JSON calls with precise parameters. This section covers the function calling protocol (tool definitions, parameter schemas, response handling), comparison across providers (OpenAI, Anthropic, Google), parallel and sequential tool calling patterns, error handling and retry strategies, and security considerations for tool access control.

3.6 RAG (Retrieval-Augmented Generation) with Agents

RAG combines retrieval systems with generation to ground Agent responses in factual, up-to-date information. The pipeline includes document chunking, embedding generation, vector storage, similarity search, and context injection. Advanced RAG techniques covered include query rewriting, multi-hop retrieval, hybrid search (dense + sparse), re-ranking, and the integration of RAG with Agent memory systems for persistent knowledge augmentation.

3.7 Multimodal Capabilities in Agents

Modern Agents increasingly leverage multimodal inputs and outputs: vision capabilities for image/screenshot understanding and OCR, audio processing for speech recognition and synthesis, video understanding for temporal event analysis, and cross-modal reasoning that combines text, images, and structured data. This section explores how multimodal perception dramatically expands Agent application scenarios from document processing to physical world interaction.

Chapter Summary

The key technologies powering AI Agents form a layered stack: LLMs as the foundation, Prompt Engineering as the interface layer, ReAct and CoT as reasoning frameworks, Function Calling as the tool bridge, RAG as the knowledge layer, and multimodal capabilities as the perception expansion. Mastering these technologies is essential for building effective Agent systems.
